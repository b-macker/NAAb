LLM Governance Engine

Drop a govern.json in your project. NAAb checks every polyglot block against your rules before execution. Three enforcement levels: HARD, SOFT, ADVISORY.

Hallucinated APIs

Catches .push() in Python, print() in JS, json.stringify() instead of json.dumps() — with "did you mean?" suggestions.

Oversimplified Code

Detects stub functions, pass-only bodies, validate() that always returns True, and fabricated status responses.

Incomplete Logic

Catches except:pass, bare raises, vague error messages, degenerate loops, and always-true conditions.

Secret Detection

Entropy-based scanning catches API keys, passwords, and tokens — even without matching known patterns.

Security Checks

SQL injection, path traversal, shell injection, privilege escalation, unsafe deserialization, and more.

CI/CD Integration

Export results as SARIF (GitHub Code Scanning), JUnit XML (Jenkins/GitLab), or JSON for custom tooling.

Governance in Action

// govern.json
{
  "version": "3.0",
  "mode": "enforce",
  "code_quality": {
    "no_oversimplification": { "level": "hard" },
    "no_incomplete_logic":    { "level": "hard" },
    "no_hallucinated_apis":   { "level": "soft" }
  }
}
